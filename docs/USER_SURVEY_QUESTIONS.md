## Research study information

ข้อมูลการวิจัย

Study Title: Evaluation of a Real-Time Badminton Training System for Shot Tracking and Performance Analytics

ชื่อการศึกษา: การประเมินระบบฝึกซ้อมแบดมินตันแบบเรียลไทม์สำหรับการติดตามการตีและการวิเคราะห์ประสิทธิภาพ

Purpose: This survey aims to evaluate the usability, perceived usefulness, and user satisfaction of the Badminton Training System. Your responses will help identify strengths and areas for improvement.

วัตถุประสงค์: แบบสำรวจนี้มีวัตถุประสงค์เพื่อประเมินความสามารถในการใช้งาน การรับรู้ประโยชน์ และความพึงพอใจของผู้ใช้ที่มีต่อระบบฝึกซ้อมแบดมินตัน คำตอบของคุณจะช่วยระบุจุดแข็งและสิ่งที่ต้องปรับปรุง

Estimated Time: 10-15 minutes

เวลาโดยประมาณ: 10-15 นาที

Confidentiality: All responses are anonymous and will be used solely for research purposes.

การรักษาความลับ: คำตอบทั้งหมดเป็นแบบไม่ระบุตัวตนและจะใช้เพื่อการวิจัยเท่านั้น

## Section 1: System Usability Scale (SUS)

ส่วนที่ 1: มาตราส่วนความสามารถในการใช้งานระบบ (SUS)

The System Usability Scale is a validated 10-item questionnaire (Brooke, 1996). Please indicate your level of agreement with each statement.

Scale: 1 = Strongly Disagree, 2 = Disagree, 3 = Neutral, 4 = Agree, 5 = Strongly Agree

มาตราส่วน: 1 = ไม่เห็นด้วยอย่างยิ่ง, 2 = ไม่เห็นด้วย, 3 = เฉยๆ, 4 = เห็นด้วย, 5 = เห็นด้วยอย่างยิ่ง

I think that I would like to use this system frequently

ฉันคิดว่าฉันอยากจะใช้ระบบนี้บ่อยๆ

I found the system unnecessarily complex

ฉันพบว่าระบบนี้มีความซับซ้อนโดยไม่จำเป็น

I thought the system was easy to use

ฉันคิดว่าระบบนี้ใช้งานง่าย

I think that I would need the support of a technical person to be able to use this system

ฉันคิดว่าฉันต้องการความช่วยเหลือจากผู้เชี่ยวชาญด้านเทคนิคเพื่อใช้งานระบบนี้

I found the various functions in this system were well integrated

ฉันพบว่าฟังก์ชันต่างๆ ในระบบนี้ถูกรวมเข้าด้วยกันได้ดี

I thought there was too much inconsistency in this system

ฉันคิดว่าระบบนี้มีความไม่สอดคล้องกันมากเกินไป

I would imagine that most people would learn to use this system very quickly

ฉันคิดว่าคนส่วนใหญ่จะเรียนรู้การใช้ระบบนี้ได้อย่างรวดเร็ว

I found the system very cumbersome to use

ฉันพบว่าระบบนี้ใช้งานยุ่งยากมาก

I felt very confident using the system

ฉันรู้สึกมั่นใจมากในการใช้ระบบนี้

## Section 2: Technology Acceptance Model (TAM)

ส่วนที่ 2: แบบจำลองการยอมรับเทคโนโลยี (TAM)

The following questions are adapted from the Technology Acceptance Model (Davis, 1989). Please indicate your level of agreement with each statement.

Scale: 1 = Strongly Disagree, 2 = Disagree, 3 = Neutral, 4 = Agree, 5 = Strongly Agree

มาตราส่วน: 1 = ไม่เห็นด้วยอย่างยิ่ง, 2 = ไม่เห็นด้วย, 3 = เฉยๆ, 4 = เห็นด้วย, 5 = เห็นด้วยอย่างยิ่ง

### Perceived Usefulness (PU)

การรับรู้ประโยชน์ (PU)

การใช้ระบบนี้ช่วยปรับปรุงประสิทธิภาพการฝึกซ้อมของฉัน

Using the system increases my training productivity

การใช้ระบบนี้เพิ่มผลิตภาพในการฝึกซ้อมของฉัน

Using the system enhances my effectiveness in training

การใช้ระบบนี้เพิ่มประสิทธิผลในการฝึกซ้อมของฉัน

Using the system makes it easier to track training progress

การใช้ระบบนี้ทำให้ติดตามความก้าวหน้าในการฝึกซ้อมได้ง่ายขึ้น

I find the system useful for badminton training

ฉันพบว่าระบบนี้มีประโยชน์สำหรับการฝึกซ้อมแบดมินตัน

### Perceived Ease of Use (PEOU)

การรับรู้ความง่ายในการใช้งาน (PEOU)

Learning to operate the system was easy for me

การเรียนรู้วิธีใช้งานระบบนี้เป็นเรื่องง่ายสำหรับฉัน

I find it easy to get the system to do what I want it to do

ฉันพบว่าการสั่งให้ระบบทำในสิ่งที่ฉันต้องการเป็นเรื่องง่าย

My interaction with the system is clear and understandable

การโต้ตอบของฉันกับระบบนั้นชัดเจนและเข้าใจได้

I find the system flexible to interact with

ฉันพบว่าระบบนี้มีความยืดหยุ่นในการใช้งาน

It was easy for me to become skillful at using the system

ฉันสามารถเชี่ยวชาญในการใช้งานระบบนี้ได้อย่างง่ายดาย

I find the system easy to use

ฉันพบว่าระบบนี้ใช้งานง่าย

### Behavioral Intention to Use (BI)

ความตั้งใจเชิงพฤติกรรมในการใช้งาน (BI)

I intend to continue using the system in the future

ฉันตั้งใจที่จะใช้ระบบนี้ต่อไปในอนาคต

I would recommend the system to other coaches/athletes

ฉันจะแนะนำระบบนี้ให้โค้ช/นักกีฬาคนอื่นๆ

I plan to use the system regularly for training sessions

ฉันวางแผนที่จะใช้ระบบนี้เป็นประจำสำหรับการฝึกซ้อม

Reference: Davis, F. D. (1989). Perceived Usefulness, Perceived Ease of Use, and User Acceptance of Information Technology. MIS Quarterly, 13(3), 319-340.

## Section 6: Net Promoter Score (NPS)

ส่วนที่ 6: คะแนนความภักดีของผู้ใช้ (NPS)

On a scale of 0-10, how likely are you to recommend the Badminton Training System to a colleague or friend?

ในมาตราส่วน 0-10 คุณมีแนวโน้มที่จะแนะนำระบบฝึกซ้อมแบดมินตันให้เพื่อนร่วมงานหรือเพื่อนมากน้อยเพียงใด?

0 = Not at all likely

0 = ไม่มีแนวโน้มเลย

10 = Extremely likely

10 = มีแนวโน้มอย่างยิ่ง

Scoring (for researcher): 0-6 = Detractor, 7-8 = Passive, 9-10 = Promoter. NPS = % Promoters - % Detractors.

การให้คะแนน (สำหรับผู้วิจัย): 0-6 = ผู้วิจารณ์, 7-8 = ผู้เฉยเมย, 9-10 = ผู้สนับสนุน NPS = % ผู้สนับสนุน - % ผู้วิจารณ์

## Survey scoring guide (for researcher)

คู่มือการให้คะแนนแบบสำรวจ (สำหรับผู้วิจัย)

### SUS score calculation

การคำนวณคะแนน SUS

For items 1, 3, 5, 7, 9: Score = Response - 1

สำหรับข้อ 1, 3, 5, 7, 9: คะแนน = คำตอบ - 1

For items 2, 4, 6, 8, 10: Score = 5 - Response

สำหรับข้อ 2, 4, 6, 8, 10: คะแนน = 5 - คำตอบ

Sum all scores and multiply by 2.5

รวมคะแนนทั้งหมดแล้วคูณด้วย 2.5

Final score range: 0-100 (>68 = above average usability)

ช่วงคะแนนสุดท้าย: 0-100 (>68 = ความสามารถในการใช้งานสูงกว่าค่าเฉลี่ย)

### TAM score calculation

การคำนวณคะแนน TAM

Perceived Usefulness (PU): Average of items 1-6

การรับรู้ประโยชน์ (PU): ค่าเฉลี่ยของข้อ 1-6

Perceived Ease of Use (PEOU): Average of items 7-12

การรับรู้ความง่ายในการใช้งาน (PEOU): ค่าเฉลี่ยของข้อ 7-12

Behavioral Intention (BI): Average of items 13-15

ความตั้งใจเชิงพฤติกรรม (BI): ค่าเฉลี่ยของข้อ 13-15

Higher scores indicate stronger agreement (range: 1-5)

คะแนนที่สูงกว่าแสดงถึงความเห็นด้วยที่มากกว่า (ช่วง: 1-5)

### NPS calculation

การคำนวณ NPS

NPS = (% of Promoters) - (% of Detractors)

NPS = (% ผู้สนับสนุน) - (% ผู้วิจารณ์)

Score range: -100 to +100

ช่วงคะแนน: -100 ถึง +100

Positive score = Good, >50 = Excellent

คะแนนบวก = ดี, >50 = ยอดเยี่ยม

### Reliability testing

การทดสอบความน่าเชื่อถือ

Calculate Cronbach's Alpha for each scale

คำนวณค่าสัมประสิทธิ์แอลฟาของครอนบาคสำหรับแต่ละมาตราส่วน

Acceptable threshold: α ≥ 0.70

เกณฑ์ที่ยอมรับได้: α ≥ 0.70
